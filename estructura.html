2. Revisando la estructura del proyecto

El proyecto que has creado está compuesto por un buen número de carpetas y ficheros que hacen que tu aplicación web funcione como la seda. Es importante que conozcas la finalidad de algunas de estas carpetas y los ficheros que contienen.

  • Carpeta app. Es la carpeta más importante, la que contiene la mayor parte del código de tu aplicación para funcionar.
  • Carpeta app/assets. Contiene ficheros estáticos que serán proporcionados a tus visitantes por parte del servidor de rails. En esta carpeta se almacenan imágenes, hojas de estilo CSS, y ficheros de código JavaScript.
  • Carpeta app/models. En esta carpeta se incluyen las definiciones en código de los modelos de tu base de datos. En el siguiente paso crearás el modelo para guardar las fotos de tu aplicación.
  • Carpeta app/controllers. Los ficheros contenidos en esta carpeta definen las acciones que se pueden realizar sobre los modelos de tu aplicación web. Por ejemplo, el controlador para el modelo de las fotos permitirá visualizar esas fotos, crear nuevas fotos, o borrarlas.
  • Carpeta app/views. Contiene ficheros con código HTML, que conforman las diferentes vistas de tu aplicación web.
  • Carpeta db. Esta carpeta contiene todos los datos de tu aplicación web. Ahora mismo la carpeta está vacía, pero en el siguiente paso empezará a llenarse con ficheros de código con información sobre las fotos de la aplicación.
  • Fichero Gemfile. ¿Recuerdas cuando estabas creando la aplicación en el paso 1 y tuviste que esperar a que se descargaran dependencias? Todas esas dependencias son librerías llamadas gemas que pueden ser descargadas de internet. Si abres el fichero Gemfile, podrás ver que no es más que una lista con las gemas que se instalan con tu aplicación.
